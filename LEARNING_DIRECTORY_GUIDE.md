# Learning 目录自动标签功能说明

## 功能说明

现在 Technology 下拉菜单会自动读取 `src/content/learning/` 目录下的所有子目录，并动态生成标签选项。

## 使用方法

### 1. 添加新目录

在 `src/content/learning/` 目录下创建新的子目录，例如：

```
src/content/learning/
  ├── AI/
  ├── Math/
  ├── Tools/
  └── NewCategory/  ← 新建目录
      └── 文章.md
```

### 2. 为新目录下的文章添加标签

**重要**：为了让标签筛选功能正常工作，你需要在新目录下的每篇文章的 frontmatter 中添加对应的标签。

例如，如果你创建了 `NewCategory` 目录，那么该目录下的文章应该这样写：

```markdown
---
title: "文章标题"
id: "2026-01-15-01"
date: "2026-01-15"
description: "文章描述"
tags: ["NewCategory", "其他标签"]  ← 必须包含目录名作为标签
---
```

### 3. 自动更新

- ✅ **下拉菜单**：会自动检测新目录并添加到下拉菜单中
- ⚠️ **标签筛选**：需要手动在文章的 frontmatter 中添加目录名作为标签

## 示例

假设你创建了一个新目录 `Web`：

1. **创建目录结构**：
   ```
   src/content/learning/Web/
     └── React入门.md
   ```

2. **在文章中添加标签**：
   ```markdown
   ---
   title: "React入门"
   id: "2026-01-15-01"
   date: "2026-01-15"
   description: "React基础教程"
   tags: ["Web", "React", "前端"]  ← 必须包含 "Web"
   ---
   ```

3. **结果**：
   - Technology 下拉菜单会自动显示 "Web" 选项
   - 点击 "Web" 会跳转到 `/learning?tag=Web`
   - 只有包含 "Web" 标签的文章会被显示

## 注意事项

1. **目录名大小写敏感**：标签名必须与目录名完全一致（包括大小写）
   - ✅ 正确：目录 `AI` → 标签 `["AI"]`
   - ✅ 正确：目录 `Math` → 标签 `["Math"]`
   - ❌ 错误：目录 `AI` → 标签 `["ai"]`（大小写不一致，无法匹配）
   - ❌ 错误：目录 `Math` → 标签 `["math"]`（大小写不一致，无法匹配）
   
   **重要**：下拉菜单显示的标签名与目录名完全一致，文章的标签也必须与目录名大小写完全一致，否则筛选功能无法正常工作。

2. **标签必须存在**：如果文章没有对应的标签，即使在下拉菜单中点击了该目录，也不会显示该文章

3. **排序**：下拉菜单中的目录按字母顺序排序（保持原始大小写）
